# AESPD Deliverables

Advanced Systems Engineering – Proposal Development.

This site is the working source-of-truth for the AESPD deliverables. The
content was migrated from a single Word document into per-deliverable
markdown files so that:

- Edits are version-controlled, diffable, and auditable
- Multiple contributors can work without overwriting each other
- A change in one deliverable can be propagated to its dependents
  through cross-links and traceable updates
- Reviewer comments are preserved as collapsible notes alongside the
  text they refer to

## How the site is organised

The deliverables are grouped by stage:

| Stage                                                          | What it covers                                          |
|----------------------------------------------------------------|---------------------------------------------------------|
| [Stage 1 – Requirements](stages/1-requirements/index.md)       | Stakeholders, needs, requirements, verification         |
| [Stage 2 – System Concept](stages/2-system-concept/index.md)   | Functional analysis, system requirements, traceability  |
| [Stage 3 – Stage Synthesis](stages/3-stage-synthesis/index.md) | Subsystem decomposition, SRD, verification requirements |
| [Stage 4 – Later Deliverables](stages/4-later-deliverables/index.md) | Remaining deliverables (to be classified)         |

Each stage's overview page lists the deliverables that belong to it; each
deliverable lives in its own page.

## Reading comments

Throughout the deliverables you'll see collapsible notes that look like
this:

??? note "Sadaf · 2026-05-06"
    Example: this is a reviewer comment preserved from the source
    document. Click the title bar to expand or collapse it.

They are review comments from the original document, kept alongside the
content they annotate.
